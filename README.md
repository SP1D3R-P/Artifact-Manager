
# a2fclt [Artifact Command Line Tool] — Artifact build and delivery toolkit

A small toolkit for building, packaging and delivering artifacts. The repository contains a builder service, a CLI, and a consumer application used to run and test artifact builds locally or in Kubernetes.

## Quick overview
- `a2fclt/` — CLI and command binaries (`a2fclt`).
- `builder/` — builder service and related internals (artifact writer, checksum, project handling).
- `consumer/` — consumer application that can receive/run produced artifacts.
- `k8s/` — Kubernetes manifests for local clusters (minikube).
- `testData/` — sample projects and test fixtures.

## Prerequisites
- Go 1.20+ installed
- Make
- (optional) Docker and Minikube for Kubernetes deployment

## Quickstart (local)
1. Build CLI and binaries:
```sh
make build
```
The `a2fclt` binary will be written to `./bin/`.

2. Run the CLI against a sample project:
```sh
./bin/a2fclt build ./testData/helloPy/
```
```sh
./bin/a2fclt build ./testData/helloC/
```
make sure point to the directory where the config.json is present 

## Quickstart (Minikube)

Ensure your terminal is pointing to the Minikube Docker daemon:
```sh
eval $(minikube docker-env)
echo $DOCKER_HOST
```
make sure that `echo $DOCKER_HOST` result is not blank 

1. Start a Minikube tunnel to expose services:
```sh
minikube tunnel
```
2. In another terminal, start Kubernetes manifests:
```sh
make k8-start
```
3. Open the builder API (default) at http://localhost:9000/
---
EndPoints : 
>> * http://localhost:9000/builds -> list artifacts
>> * http://localhost:9000/artifacts -> view the artifacts 
>> * http://localhost:9000/{build_id}/output.txt -> view the ouput generated by specific build
>> * http://localhost:9000/builds/{build_id} -> view the final config generated after processing.

## Build an artifact
Use the CLI or the builder HTTP API to request builds. Example (CLI):
```sh
./bin/a2fclt <path/to/project>
```

## Project structure
- `a2fclt/` — CLI implementation and `main` entrypoints.
- `builder/` — core builder logic: artifact handling, writers, checksum, project config and types.
- `consumer/` — consume the artifacts generated by the builder. store it & serve it.
- `k8s/` — YAML manifests for deploying builder and consumer.
